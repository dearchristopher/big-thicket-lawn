### Fix 1 (IMPLEMENTED вњ…): areaServed enumeration
**Impact: HIGH** вЂ” Helps the site rank for "lawn care Silsbee TX", "lawn mowing Kountze TX", "Beaumont lawn service" searches. Structured data with named cities improves local pack eligibility.  
**Effort: LOW** вЂ” Single JSON-LD object update in `index.html`.  
**Change:** `areaServed` expanded from a single generic Place to 5 entries (Lumberton, Silsbee, Kountze, Beaumont + Hardin County).

---

### Fix 2 (REPORTED): Dynamic JSON-LD synced from Sanity
**Impact: MEDIUM-HIGH** вЂ” Phone, hours, priceRange, and serviceType are all hardcoded. If any changes in Sanity (new service added, new hours), JSON-LD becomes stale and may mislead Google.  
**Effort: MEDIUM** вЂ” Requires a new React component that reads `SiteSettingsContext` and injects a `<script type="application/ld+json">` via `dangerouslySetInnerHTML`.  

**Recommended approach:**
1. Create `src/components/LocalBusinessJsonLd.tsx` that consumes `useSiteSettingsContext()`
2. Build the JSON-LD object from Sanity fields: `phoneNumber`, `businessHours`, `serviceArea`, `facebookPageUrl`, `yelpReviewUrl`, plus active `service` names
3. Mount in `App.tsx` (inside `SiteSettingsProvider`) вЂ” removes static script from `index.html`
4. Keep `index.html` static script as SSR fallback skeleton with current values, or remove entirely

---

### Fix 3 (REPORTED): SmugMug hero carousel fallback images are original/4K resolution
**Impact: MEDIUM** вЂ” Only affects preview deploys and Sanity-down scenarios; production users get Sanity gallery images. On preview deploys, 4K/original images (2вЂ“15 MB) load in the hero carousel, tanking LCP.  
**Effort: LOW-MEDIUM** вЂ” Need to verify correct XL-size URLs via SmugMug UI before substituting.  

**Recommended action for owner/dev:** In SmugMug, navigate to each fallback image and copy the XL size share URL. Then update `src/components/HeroGalleryCarousel.tsx` lines 14вЂ“29 `DEFAULT_GALLERIES` array. Alternatively, upload preferred fallback images to Sanity as gallery docs so the hardcoded fallbacks are rarely needed.
